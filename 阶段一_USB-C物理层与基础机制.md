# 阶段一：USB-C 物理层与基础机制

> 学习目标：掌握 USB-C 引脚定义、CC 机制、角色体系，为理解 PD 协议打下基础。

## 1.1 USB-C 接口引脚定义（引脚级）

USB-C 连接器共 24 个引脚，正反面（翻转后）功能对称，因此正插反插都能用。

```
        A 面（正插）                    B 面（反插）
┌──────────────────────────┐
│ A1   GND                  │  B12  GND
│ A2   TX1+  ─┐ SuperSpeed  │  B11  RX1+
│ A3   TX1-  ─┘ TX 差分对   │  B10  RX1-
│ A4   VBUS                  │  B9   VBUS
│ A5   CC1  ◀── 核心！       │  B8   SBU2
│ A6   D+   ─┐ USB 2.0      │  B7   D-
│ A7   D-   ─┘              │  B6   D+
│ A8   SBU1                  │  B5   CC2  ◀── 核心！
│ A9   VBUS                  │  B4   VBUS
│ A10  RX2+  ─┐ SuperSpeed  │  B3   TX2+
│ A11  RX2-  ─┘ RX 差分对   │  B2   TX2-
│ A12  GND                  │  B1   GND
└──────────────────────────┘
```

### 重点引脚速查表

| 引脚 | 功能 | 重要性 |
|------|------|--------|
| **CC1 / CC2** | Configuration Channel（配置通道） | ⭐⭐⭐ 核心！PD 协议唯一通信通道 |
| **VBUS** | 电源总线（最高 48V/5A） | ⭐⭐ 实际供电通道 |
| **TX+/TX- / RX+/RX-** | SuperSpeed 差分对（USB 3.x / USB4） | ⭐⭐ 高速数据传输 |
| **D+ / D-** | USB 2.0 差分对 | ⭐ 向下兼容 USB 2.0 |
| **SBU1 / SBU2** | Sideband Use | ⭐ Alt Mode 时用（如 DP 音频） |

## 1.2 CC 引脚——USB-C 的神经中枢

CC 引脚同时承担三项功能：

### 第一功能：检测插入方向

USB-C 能正插反插都识别，靠 CC1/CC2 的组合：

```
正插时：
  Source 的 CC1 ── 连接 ──▶ Sink 的 CC1（有 Rd 下拉）
  Source 的 CC2 ── 悬空 ──▶ Sink 的 CC2（悬空）

反插时：
  Source 的 CC1 ── 悬空
  Source 的 CC2 ── 连接 ──▶ Sink 的 CC1（有 Rd 下拉）
```

Source 通过检测**哪个 CC 引脚有下拉电阻**，既知道插入方向，又知道用哪个 CC 引脚通信。

---

### 第二功能：识别对端角色

靠 **Rp（上拉电阻）** 和 **Rd（下拉电阻）** 的组合：

| 端类型 | CC 引脚配置 | 说明 |
|--------|-------------|------|
| Source | CC 接 Rp（上拉到 VBUS 或 3.3V） | 提供电源 |
| Sink | CC 接 Rd（下拉到 GND，标准值 5.1kΩ） | 消耗电源 |
| DRP（Dual Role Power） | 动态切换 Rp/Rd | 可供电可受电（笔记本、手机） |

**Rp 的阻值还编码了默认电流能力：**

| Rp 阻值 | 默认电流 | 含义 |
|---------|---------|------|
| 56kΩ | 500mA / 900mA | USB 标准默认电流 |
| 22kΩ | 1.5A | 中等充电能力 |
| 10kΩ | 3A | 高电流充电能力 |

> 💡 **这就是为什么有些不支持 PD 协议的充电器，仍然能快充** —— 靠 Rp 阻值就能告诉 Sink "我能给 3A"，完全不需要 PD 协议参与。

---

### 第三功能：传输 PD 协议消息

当双方都支持 USB Power Delivery 时，CC 引脚切换为**通信模式**，用 **BMC（Bi-phase Mark Coding）** 编码传输 PD 消息。

**BMC 编码原理：**

```
数据 0：bit 周期内不翻转
数据 1：bit 周期内翻转一次

例：传输 0b10110
     ─┐ ┌─┐ ┌──┐ ┌─
      └─┘ └─┘  └─┘
      1  0  1   1   0
```

**BMC 编码的特点：**

- 自带时钟同步，不需要额外的时钟线
- 对 CC 线阻抗变化不敏感
- 速率 300kHz（低速），对线缆质量要求低
- 单根 CC 线同时承载双向通信（全双工）

## 1.3 VBUS 电源总线

VBUS 是**实际供电**的通道，但它**不传任何协议消息**。

所有 "我要多少电"、"我能给多少电" 都在 CC 引脚上谈好，谈妥之后 VBUS 才按约定的电压/电流供电。

**VBUS 功率演进：**

| 规范版本 | 最高电压 | 最大电流 | 最大功率 |
|---------|---------|---------|---------|
| USB 2.0 | 5V | 500mA | 2.5W |
| USB Battery Charging 1.2（BC1.2） | 5V | 1.5A | 7.5W |
| USB PD 2.0 | 20V | 5A | 100W |
| USB PD 3.1 SPR（标准功率范围） | 20V | 5A | 100W |
| **USB PD 3.1 EPR（扩展功率范围）** | **48V** | **5A** | **240W** |

> ⚠️ **EPR（Extended Power Range，48V/240W）是当前最前沿的战场。**
> 48V 对连接器、线缆、PCB 走线、保护电路都是全新的挑战。
> 连接器热失控、线缆载流瓶颈，都是还未被彻底解决的问题。
> 这也是「USB-C 热崩溃防护」专利方向的行业背景。

## 1.4 USB-C 的角色体系

USB-C 有三组独立又相互配合的角色体系，搞清楚这些是理解互操作性问题的基础：

```
┌─────────────────────────────────────────────────────────┐
│  供电角色（Power Role）                                  │
│  ┌─────────┐  提供 VBUS 电源                             │
│  │ Source  │                                            │
│  └────┬────┘                                            │
│       ↕ PR_Swap（Power Role Swap）                     │
│  ┌────┴────┐                                            │
│  │  Sink   │  消耗 VBUS 电源                             │
│  └─────────┘                                            │
│       ↕ DRP（Dual Role Power，可切换）                   │
│  ┌─────────┐                                            │
│  │  DRP    │  可作为 Source 也可作为 Sink               │
│  └─────────┘  （笔记本、手机、显示器都是 DRP）            │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  数据角色（Data Role）                                    │
│  ┌─────────┐  USB Host（下行端口）                       │
│  │   DFP   │                                            │
│  └────┬────┘                                            │
│       ↕ DR_Swap（Data Role Swap）                       │
│  ┌────┴────┐                                            │
│  │   UFP   │  USB Device（上行端口）                    │
│  └─────────┘                                            │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  重要原则：供电角色和数据角色是各自独立的！              │
│                                                         │
│  笔记本接显示器：笔记本 = Source（供电）+ DFP（Host）   │
│  手机反向给笔记本充电：手机 = Source（供电）+ UFP（Device）│
└─────────────────────────────────────────────────────────┘
```

**三种端口类型的常见设备：**

| 端口类型 | 说明 | 典型设备 |
|---------|------|---------|
| Source-only | 只能供电 | 充电器、壁挂电源 |
| Sink-only | 只能受电 | 手机（充电时）、耳机 |
| DRP | 可供电可受电 | 笔记本、双向充电宝 |
| DRD | 可切换数据角色 | 支持 DisplayPort 的显示器 |
